# Правила работы с ассистентом

## 1. Коммитит и пушит только пользователь

Ассистент **не выполняет** `git add`, `git commit`, `git push`. Он:

- правит файлы в рабочей копии;
- запускает локальные проверки (typecheck, тесты, линт);
- формулирует текст commit-message и список изменённых файлов;
- останавливается перед фазой commit и передаёт управление пользователю.

Исключение — пользователь явно просит ("закоммить и запуш", "сделай commit + push"). Без явного запроса ассистент не трогает git-историю.

## 2. Резюме завершённой работы — в виде commit-message

После завершения задачи ассистент **не пишет** длинное "## Готово!"-резюме в чат. Вместо этого выдаёт **проект коммита** в формате:

```
Файлы:
 - path/to/foo.ts — что и зачем
 - path/to/bar.tsx — что и зачем

Предлагаемый commit message:

<тип>(<область>): краткий заголовок ≤ 72 символов

Тело — почему и что (3–5 строк).
```

Заголовок — conventional-стиль (`feat(...)`, `fix(...)`, `docs(...)`, `refactor(...)`).

**Всегда один коммит на все изменённые файлы** — пользователь грузит их одним пушем. Даже если изменения логически разнородны, ассистент **не предлагает** разбивать их на несколько коммитов; вместо этого пишет один общий заголовок (с самым важным типом изменения) и в теле перечисляет логические блоки маркерами.

## 3. Маскировка секретов в командах

Любая команда, которая может вывести секрет (PEM, пароль, API-ключ, env-значение, JWT) **должна явно маскировать вывод**:

- `wc -c file` / `stat -c %s file` — длина, не содержимое;
- `md5sum` / `sha256sum` — fingerprint вместо тела;
- `head -c 12 ; echo '...'` — префикс с маской;
- `openssl pkey -in file -noout` (без `-text`) — exit-code, не raw байты.

Запрещены без явной маскировки на путях `*.env`, `*.pem`, `/srv/matcheck/secrets/*`: `cat`, `echo $VAR`, `env`, `printenv`, `set`, `tee` без `>/dev/null`.

`openssl rand` — всегда `> file`, никогда `| tee` в stdout.

В чат пользователь копирует **только** маскированную часть (`M***Bx`), не реальные значения. Если секрет случайно попал в чат — ротировать (для PG — пароль в YC Console; S3 — отозвать access key; JWT/AES — перевыпустить).

## 4. Язык общения — русский

Ассистент **всегда** пишет на русском языке: и в ответах пользователю, и во внутренних размышлениях (thinking), и в комментариях к плану/коду/коммиту. Английский допустим только в технических идентификаторах (имена функций, переменных, файлов, опции CLI) и в цитатах из чужого кода/документации.
